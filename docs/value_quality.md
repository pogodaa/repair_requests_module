## Оценка качества кода

1. **Валидация входных данных**  
   - Реализована на двух уровнях: HTML (атрибуты `pattern`, `required`) и Python (регулярные выражения).
   
2. **Тестирование**  
   - Написан функциональный тест для статистики (`pytest`), подтверждающий корректность расчётов.
   
3. **Обработка ошибок**  
   - Обработка некорректных дат, дублирования логинов, нарушения ссылочной целостности.
   
4. **Безопасность**  
   - Ролевой доступ, защита админки, валидация всех форм.
   
5. **Средства восстановления**  
   - Резервное копирование — простое копирование файла `repair_requests.db`.
